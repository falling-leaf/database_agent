# Files Outside Git

This document lists all directories and files in `/home/why/dbagent/` that are **not tracked** by git.
These are excluded via `.gitignore` because they are either too large, build artifacts, or machine-specific data.

Total disk usage (untracked): **~64 GB**

---

## 1. `pg_base/` -- PostgreSQL Installation + Database Cluster (21 GB)

| Subdirectory | Size | Description |
|---|---|---|
| `pg_base/data/` | 21 GB | PostgreSQL data directory (database cluster, WAL logs, tablespace files) |
| `pg_base/bin/` | 13 MB | PostgreSQL binaries (psql, pg_ctl, postgres, etc.) |
| `pg_base/lib/` | 16 MB | Shared libraries including `postgresql/pgdl.so` and `postgresql/pgdl/` |
| `pg_base/include/` | 6.4 MB | PostgreSQL C headers |
| `pg_base/share/` | 2.8 MB | PostgreSQL shared files, extension SQL scripts |
| `pg_base/home/` | 9.9 MB | Nested DESTDIR artifact from `compile.sh` `make install` |

**Rebuild from source:** Run `bash postgresql_install.sh` (builds PostgreSQL 13.22 from `pgdl/third_party/postgresql-13.22/`).

**How to restore:** If you lose `pg_base/`, run `bash postgresql_install.sh` to rebuild PostgreSQL and reinitialize the data directory. Then run `bash compile.sh` to rebuild the pgdl extension. Note: the database data itself (tables, rows) will be lost -- only the binaries are reproducible.

## Summary: Recovery Order

If you need to rebuild this project from scratch (git clone + restore):

```
1. pgdl/shell/download_build_thirdparty.sh   # downloads libtorch, PostgreSQL source, etc.
2. bash postgresql_install.sh                 # builds PostgreSQL, initializes data dir
3. bash compile.sh                            # builds pgdl.so extension
4. pgdl/test/prepare_models.py                # downloads ML model weights
5. (datasets must be obtained separately -- see data/ subdirectories)
```
